## Important — about the "missing expected content" findings

Of the 11 medium issues, ~9 are flagged as **"page X missing expected content '<keyword>'"**. After running the harness, I manually verified several of these via curl with the same cookies the harness uses:

```
curl -H "Cookie: site_gate=1; demo_user=…" http://localhost:3000/companies
→ contains "compan" (the H1)
curl -H "Cookie: site_gate=1; demo_user=…" http://localhost:3000/portal/feedback
→ contains "feedback" / "recommend"
```

The **content is there**. The harness uses `page.locator('body').innerText()` after a 1500 ms wait — combined with Next.js dynamic-import streaming and React hydration timing, that wait is too short on this dev server, so the body text it sees is the loading skeleton (no page-specific keywords). These are **harness timing false positives**, not real bugs.

**Action item:** when reading the per-issue list below, skim past "missing expected content" findings unless the page genuinely doesn't exist (404). The remaining genuine signals to triage are noted in the **"Real findings worth a manual check"** section below.
